# TodoForest — AI-планер дня

## Ідея

Todoist-подібний планер, де юзер не розкладає задачі вручну по папках і датах, а
просто вивалює все, що в голові (текстом або голосом) в одне поле. AI сам
перетворює хаос на структуровані задачі — з пріоритетом, датою і часом — і
щодня формує чіткий план на сьогодні з накопиченого беклогу.

## Цінність

Головна біль Todoist та подібних тулів — high friction на вхід: щоб додати
задачу правильно, треба самому проставити проєкт, дату, пріоритет, теги.
Люди або кидають це на середині, або звалюють усе в Inbox і ніколи не
розбирають. TodoForest прибирає цей крок: користувач говорить/пише як думає
("завтра подзвонити в банк, доробити презентацію до п'ятниці, терміново
відповісти клієнту"), а AI сам розкладає це на задачі з датами і пріоритетом.

## Користувацький флоу (3 екрани)

1. **Capture** — поле "Що в голові?" + мікрофон. Один вхід може містити
   кілька задач одразу, диктуємо все підряд без структури.
2. **Inbox** — увесь беклог структурованих задач, які AI розпарсив із
   інпутів: назва, пріоритет, дедлайн, час. Тут можна вручну поправити
   помилки AI.
3. **Today** — список задач на сьогодні, сформований з беклогу за датою
   `dueDate === сьогодні` (+ протерміновані). Чекбокси, відмітка виконання.

## Скоуп MVP (що точно є)

- Текстовий Capture з парсингом через Claude (обов'язково для здачі).
- AI коректно розуміє "сьогодні / завтра / післязавтра / у понеділок / за
  дату" відносно **реальної поточної дати**, яку сервер підставляє в промпт
  (а не вигадує модель).
- Один вхід → кілька задач, кожна з полями: title, priority, dueDate, time.
- Inbox з можливістю ручного редагування дати/пріоритету, відмітки
  виконання, видалення.
- Today — задачі на сьогодні + протерміновані, з чекбоксами.
- Порожній стан для нового користувача (не виглядає як баг).
- Мобільна верстка (mobile-first, one-handed).
- Дані зберігаються в браузері (localStorage) — без бекенд-БД, це навмисне
  спрощення MVP.

## Скоуп поза MVP (advanced, за наявності часу)

Голосове введення (Web Speech API поверх текстового поля), week-вигляд,
сторінка деталей задачі, фільтри за тегами, drag-and-drop, перенесення
невиконаного на завтра, інтеграція з Google Calendar, Telegram-бот.

## Схема задачі (контракт AI ↔ інтерфейс)

```json
{
  "id": "uuid",
  "title": "string",
  "priority": "high | medium | low",
  "dueDate": "YYYY-MM-DD | null",
  "time": "HH:MM | null",
  "notes": "string | null",
  "done": "boolean",
  "createdAt": "ISO timestamp"
}
```

## Функціональні тези

- `/api/parse` — серверний ендпоінт, приймає сирий текст + поточну дату
  (ISO, часовий пояс клієнта), викликає Claude (`anthropic/claude-haiku-4.5`)
  через OpenRouter з промптом, що просить повернути **тільки JSON-масив**
  задач за схемою вище. Дата/час обчислюються на сервері й передаються в
  промпт як "сьогодні = 2026-07-19 (неділя)" — модель не вгадує дату сама.
- Ключ OpenRouter API живе тільки в env-змінній Vercel (`OPENROUTER_API_KEY`),
  ніколи в клієнтському коді.
- Відповідь моделі не довіряємо наосліп — парсимо JSON, валідуємо поля,
  чистимо сміття, якщо модель повернула зайвий текст навколо JSON.
