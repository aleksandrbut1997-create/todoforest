# Джура — тест-чекліст перед здачею

Проходити на проді (todoforest-a.vercel.app): раз на телефоні (Chrome або
Safari), раз на десктопі (Chrome). Для чистого стану — інкогніто-вікно.
Позначай: ✅ ок / ❌ баг (і кинь Клоду опис + скріншот).

## 1. Онбординг і профіль

- [x] Перший візит (інкогніто): показується онбординг із 3 картками (Кажи / Обоз / Похід) ✅
- [x] «До діла, джуро!» → екран «Ставай до лав» ✅
- [x] «Пропустити — сам розберуся» → одразу застосунок; після перезавантаження онбординг більше не з'являється ✅
- [x] «Вписатись у реєстр» з ім'ям → тост «Вітаю в лавах, …», ім'я золотим у хедері ✅ (баг «завелика кнопка на моб» — виправлено, перевірити після деплою)
- [x] Кнопка «Вписатись у реєстр» неактивна, поки ім'я порожнє ✅
- [x] «Потім — спершу до справ» → застосунок, у хедері «Увійти» ✅
- [x] Тап по імені в хедері → екран редагування, зміна імені зберігається ✅
- [x] «Вийти з реєстру» → тост, у хедері знову «Увійти», задачі НЕ зникли ✅

## 2. Кажи (текстовий парсинг) — критично для здачі

- [ ] «завтра на курінь їхати, шаблі поточити — терміново, у п'ятницю гінця слати» → 3 задачі: завтра = 2026-07-20, п'ятниця = 2026-07-24, «терміново» → пріоритет «горить»
- [ ] «сьогодні», «післязавтра», «у понеділок» → правильні дати відносно реальної (не вигадані)
- [ ] Час розпізнається: «завтра о 10 подзвонити в банк» → чип «завтра · 10:00»
- [ ] Беззмістовний текст («привіт як справи») → помилка «не розчув жодної задачі», нічого не додано
- [ ] Порожнє поле → кнопка «Джуро, розбери!» неактивна
- [ ] Під час розбору: спінер + «Слухаю, отамане…», текстове поле заблоковане
- [ ] Після розбору: перекидає в Обоз, тост «Джура записав задач: N», нові картки блимають малиновим

## 3. Голос (Chrome/Safari; у Firefox мікрофона нема — це очікувано)

- [ ] Тап мікрофона → браузер питає дозвіл → після дозволу кнопка пульсує
- [ ] Диктування українською → текст з'являється в полі наживо, під час паузи фіксується
- [ ] Повторний тап → запис зупиняється, текст лишається, можна редагувати руками
- [ ] Заборонити доступ до мікрофона → зрозуміле повідомлення (не мовчазна поломка)
- [ ] iPhone: мікрофон працює в Safari, або кнопки нема, якщо API недоступний — без білого екрана

## 4. Обоз

- [ ] Сортування: спершу «горить», потім «до діла», потім «потерпить»
- [ ] Тап по картці → відкривається редагування дати і пріоритету; зміни живуть після перезавантаження
- [ ] Чекбокс → задача їде в «Виконано (N)» унизу; секція розгортається/згортається
- [ ] ✕ → тост «Задачу викреслено · Повернути»; «Повернути» повертає задачу на місце
- [ ] Тост зникає сам за ~5 секунд
- [ ] Порожній Обоз (новий юзер): екран «В обозі порожньо» з кнопкою в Кажи

## 5. Похід

- [ ] Задача з датою сьогодні → секція «Сьогодні», відсортовано за часом (без часу — внизу)
- [ ] Задача з минулою датою → секція «Протерміновані — наздогнати», червоний чип дати
- [ ] Виконана сьогоднішня задача → секція «Виконано сьогодні»
- [ ] Бейджі на вкладках Обоз/Похід відповідають реальній кількості невиконаних
- [ ] Порожній Похід: «Сьогодні походу нема» з кнопкою в Кажи

## 6. Мобільне і загальне

- [ ] Однією рукою: всі кнопки досяжні, по чекбоксах і ✕ не промахуєшся
- [ ] iPhone: нижня панель вкладок не перекрита системним жестом (safe-area)
- [ ] Хедер липкий при скролі довгого списку
- [ ] Перезавантаження сторінки → задачі і профіль на місці
- [ ] Інше інкогніто-вікно → чистий стан (очікувано: дані живуть на пристрої)
- [ ] Favicon зі шаблями видно у вкладці браузера
- [ ] Прод-парсинг працює (= ключ OpenRouter у Vercel живий)
